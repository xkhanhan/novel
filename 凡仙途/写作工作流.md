# 凡仙途·写作工作流

> 三句口令触发完整写-审-记闭环

---

## 标准口令

| 口令 | 触发动作 | 对应旧Agent |
|------|---------|------------|
| **"写第X章"** | 读取章纲+前文摘要+角色状态→生成写作任务书→写出正文 | Context Agent |
| **"审查"** | 对照审查清单.md 做6维检查→输出问题清单 | Reviewer |
| **"更新记录"** | 更新 state.json + 摘要 + 事件日志 | Data Agent |

---

## 完整闭环

```
你说："写第13章"
  ↓
我自动：Read 第13章章纲 + 前文摘要 + 主角状态 + 相关设定
  ↓
我自动：输出"本章要点"（角色/地点/时间/目标/限制）
  ↓
我自动：写出第13章正文
  ↓
你通读后说："审查"
  ↓
我自动：按审查清单.md 逐项检查6个维度
  ↓
我自动：输出问题清单（阻塞/高优/中低）
  ↓
你确认无问题后说："更新记录"
  ↓
我自动：更新 state.json 进度 + 写章节摘要到 .webnovel/summaries/
  ↓
完成
```

---

## 三个命令的详细职责

### 1. 写第X章（替代 Context Agent）

写前自动加载：
- `大纲/第N卷-章纲` 中对应章节目标
- 前2章正文结尾（钩子承接）
- 本次涉及的角色设定片段（从 设定集/ 读取）
- 当前境界力量体系限制（从 力量体系.md 读取）

输出：
- **本章要点**：角色+地点+时间线+目标
- **正文**：符合设定的一致性章节

### 2. 审查（替代 Reviewer）

检查范围（见审查清单.md）：
- 设定一致性、时间线、叙事连贯、角色一致性、逻辑、AI味

### 3. 更新记录（替代 Data Agent）

脚本命令：
```powershell
cd <项目根目录>
$env:PYTHONPATH='webnovel-writer\scripts'
python -m data_modules.webnovel --project-root "凡仙途" where
python -m data_modules.webnovel --project-root "凡仙途" preflight
python -m data_modules.webnovel --project-root "凡仙途" state get-progress
python -m data_modules.webnovel --project-root "凡仙途" index stats
```

---

## 注意事项

- **写前不要手动喂设定**——我会自动取，但你也可以附加说明："这次重点注意XX设定"
- **审查后你不一定全改**——我会标注严重程度，你决定哪些修、哪些忽略
- **更新记录只需要在你确认后才做**——不自动触发，避免覆盖
